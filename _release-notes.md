## v0.79 更新内容

### 完整双语（中/英）UI 支持
- 新增集中式国际化模块 `src/core/i18n/lang.ts`，统一管理所有 UI 字符串
- 覆盖侧边栏、设置面板、聊天面板、输入框、消息气泡、子智能体提示、关闭对话框等全部界面
- 语言偏好持久化存储至 SQLite，通过 `useLang()` React Hook 实现组件级响应式切换
- 设置面板新增语言选择器（中文 / English），切换后即时生效无需重启
- 子智能体提示词、工具返回标签等后端文本同步双语化

### 系统托盘支持
- 新增系统托盘图标，窗口关闭时可最小化到托盘后台运行
- 托盘右键菜单支持「显示 Codem / 退出」和「显示 / 退出」双语切换
- `update_tray_language` 命令实现托盘菜单语言随 UI 语言实时同步
- 支持 `hide_to_tray` / `show_from_tray` / `quit_app` 后端命令

### 关闭窗口行为选择
- 新增 `CloseConfirmDialog` 关闭确认对话框，支持三种行为：
  - **每次询问**：关闭时弹出选择窗口
  - **最小化到系统托盘**：后台运行，点击托盘图标恢复
  - **直接关闭程序**：完全退出
- 选择可记住，后续在设置面板中可修改

### NSIS 安装包双语语言选择器
- NSIS 安装包支持简体中文 + English 双语安装界面
- 安装时弹出语言选择器，用户可选择安装语言
- `get_installer_default_lang` 命令通过 Windows 注册表自动检测安装类型（NSIS→中文 / MSI→英文），首次启动自动设置默认语言
- `tauri.conf.json` 配置 `displayLanguageSelector: true`

### 终端面板增强
- 支持 `Ctrl+V` 从剪贴板粘贴命令（多行自动转单行）
- 支持 `Ctrl+C` 复制选中文本（无选中时保持原有中断信号行为）
- 支持右键粘贴
- 命令执行期间锁定输入，防止并发冲突

### 确定性步骤进度（对标业界方案）
- 放弃不可靠的 LLM 文本标记（`[STEP:x/y]`），改为从 AgenticLoop 迭代计数器获取 100% 准确的进度
- 采用轻量级启发式步骤估算（`estimateSteps`），零延迟、无额外 LLM 请求，彻底消除并发冲突
- 底部居中展示胶囊形进度条（`第1/3步`），支持 Hover 弹出包含圆环指示器的完整执行计划
- 纯 CSS `:hover` 实现悬浮窗，避免 React 流式渲染时事件丢失

### Codex 风格子智能体执行视图
- 子智能体详情页重构为 Codex 风格：显示"已处理"实时计时器 + 活动列表
- 活动列表实时追踪每次思考（"正在思考"→"已运行思考 3s"）和工具调用（"正在读取文件"→"已运行读取文件 1s"）
- 工具名称中文化映射（read_file→读取文件、write_file→写入文件等）
- `requestAnimationFrame` 驱动计时器，流式输出期间平滑更新

### 主窗口执行计时器
- 流式输出期间显示"已处理 Xs"计时器，`requestAnimationFrame` 直接操作 DOM 避免跳秒

### 思考过程修复
- 修复流式输出时 Reasoning 内容丢失问题（`reasoning_delta` 自动创建消息）
- 针对 DeepSeek 模型在 API 请求层注入强制中文思考指令
- 修复多迭代场景下思考过程和回复被拆分为多条消息的问题（改为单消息累积）

### 工具调用顺序修复
- 修复工具在 UI 中显示顺序错乱的问题（`wait_for_subagent` 跑到 `spawn_subagent` 前面等）
- 根因：执行器并发/串行重排序导致二次 `tool_start` 事件顺序与 LLM 输出不一致
- 修复：`tool_start` 延迟到 `tool_use_end` 发出（携带完整 input），执行阶段不再重复发 `tool_start`

### 性能优化
- 移除事件循环中的 `await import()` 动态导入，消除流式输出阻塞
- 子智能体状态轮询频率提升至 500ms，确保活动列表实时更新

## v0.77 更新内容

### 子智能体调用后主任务思考过程变为英文的修复
- 根因分析：5 个英语污染源叠加导致 LLM 语言惯性偏移
- 修复 `spawn_subagent` / `wait_for_subagent` 工具返回文本中文化（标签用中文，数据值保持英文避免编码问题）
- 修复子智能体系统提示词全面中文化（身份、任务执行、编码规则等 4 个区块）
- 修复子智能体 Agent 定义 prompt 中文化（explore/plan/general 三个角色）
- 修复 `parseTaskResult` 解析标记兼容中英文双语，fallback 默认值中文化
- 修复 `spawner.ts` 工具结果拼接标记中文化

### 主任务思考过程全英文问题修复
- 根因：`prompt.ts` 反引号修复后重新编译，新增 65 行英文编码规则导致系统提示词 95% 英文
- 在系统提示词最末尾追加强力中文语言规则段（利用 LLM recency bias）
- 包含抗英文上下文干扰指令和自我纠偏指令

### 工具调用窗口子智能体名称显示修复
- `MessageBubble.tsx` 中提取子智能体名字的正则从 `Sub-agent "..."` 更新为兼容中英文 `子智能体 "..."` / `Sub-agent "..."`

### 代码清理
- 清理代码注释中所有对标产品名称（Codex、Claude Code 等），改为中性表述
- 修复 `prompt.ts` 中多处未转义反引号导致模板字符串断裂的编译错误
- 修复测试文件中类型安全问题

### 编码安全策略
- 工具返回中：标签中文化（`状态:`、`摘要:`、`输出:`、`文件:`），数据值保持英文
- 系统提示词中：命令示例保持英文
- 工具标识符（`SUBAGENT_TASK_ID:`、`glob`、`grep`）保持英文
