# Phase B-F 功能手动测试指南

> 本文档详细说明每个新功能的操作步骤和预期结果，供开发者逐项验证。

---

## 前置准备

1. 确保已安装依赖：`npm install`
2. 启动开发模式：`npm run tauri dev`（桌面端）或 `npm run dev`（浏览器端）
3. 在设置中配置好 LLM API Key（设置 → API 配置）
4. **知识笔记本功能依赖 Embedding API**，需在 设置 → 多模态能力 中配置 Embedding Provider

---

## Phase B：工具/技能管理增强

### B1. 技能管理面板 — 基本操作

**入口**：侧边栏点击 📚「技能」按钮

#### 测试 1：查看已安装技能列表
1. 点击侧边栏 📚 按钮
2. 预期：弹出技能管理弹窗，默认显示「技能管理」tab
3. 预期：列表中显示内置技能（conversation-to-prompt、prompt-optimization、interactive、skill-creator、mermaid-diagram）
4. 预期：每个技能卡片显示名称、描述、来源标签（builtin）

#### 测试 2：搜索技能
1. 在技能管理弹窗顶部的搜索框输入「prompt」
2. 预期：列表实时过滤，只显示名称/描述包含「prompt」的技能
3. 清空搜索框，预期：列表恢复显示全部技能

#### 测试 3：启用/禁用技能
1. 找到某个技能，点击启用/禁用开关（Switch 组件）
2. 预期：开关状态切换，且状态持久化（关闭弹窗再打开，状态保持）
3. 禁用某个技能后，在聊天中发送消息触发该技能
4. 预期：被禁用的技能不会被激活

#### 测试 4：删除技能（仅非 builtin 技能）
1. 如果有用户安装的技能，点击删除按钮
2. 预期：弹出 AlertDialog 确认对话框
3. 点击「取消」→ 技能不被删除
4. 再次点击删除 → 点击「删除」→ 技能从列表消失

#### 测试 5：ZIP 上传安装技能
1. 准备一个技能 ZIP 包（内含 SKILL.md 的压缩文件）
2. 在技能管理弹窗中，将 ZIP 文件拖拽到上传区域
3. 预期：显示上传进度，安装完成后技能出现在列表中
4. 如果技能已存在，预期：弹出覆盖确认对话框

### F5. 笔记本设置

#### 测试 34：配置笔记本参数
1. 打开 设置（侧边栏 ⚙️）
2. 找到笔记本设置区域
3. 预期：显示以下配置项：
   - 分块大小（maxChunkSize，默认 2000）
   - 重叠大小（overlapSize，默认 200）
   - Top-K 结果数（topK，默认 5）
   - 相似度阈值（similarityThreshold，默认 0.3）
4. 修改某个值（如 topK 改为 10）
5. 关闭设置弹窗，重新打开
6. 预期：修改后的值被持久化保存

### F6. Embedding 配置（知识检索前提条件）

#### 测试 35：配置 Embedding Provider
1. 打开 设置 → 找到「🎨 多模态能力」
2. 点击「🎨 多模态设置」按钮
3. 预期：弹出多模态设置面板
4. 在 Embedding 区域配置：
   - Provider ID（如 openai）
   - API Key
   - Base URL（如 https://api.openai.com/v1）
   - Model（如 text-embedding-3-small）
   - 启用开关打开
5. 保存配置
6. 预期：添加来源时能成功生成向量嵌入

> **如果未配置 Embedding**：添加来源时索引会失败，来源状态显示「failed」，错误信息提示需要配置 Embedding Provider。

## 常见问题

**Q: 技能市场打不开 / 列表为空？**
A: 技能市场需要通过 Tauri Rust 代理访问 GitHub API。请确保在 Tauri 桌面端（`npm run tauri dev`）中测试，而非浏览器模式。检查网络连接是否正常。

**Q: 添加来源后索引失败？**
A: 索引需要 Embedding API。请在 设置 → 多模态能力 中配置 Embedding Provider（API Key、Base URL、Model），并确保启用。

**Q: 笔记本对话模式没有引用来源？**
A: 确保笔记本中已有已索引的来源（来源状态为「已索引」），且 Embedding 配置正确。AI 通过语义搜索检索相关内容，如果来源未索引则无法检索。

**Q: 交互式表单/提示词优化没有触发？**
A: 这些是 AI 自主决策的技能。AI 会根据用户消息内容判断是否需要加载技能。尝试更明确的指令，如「请优化我的系统提示词」或「请用表单收集我的需求」。

**Q: PDF 文件无法解析？**
A: PDF 提取使用纯 TypeScript 实现，支持标准 PDF 格式。某些加密或特殊编码的 PDF 可能无法正常提取。

**Q: 本地嵌入模型下载失败？**
A: 默认模型（all-MiniLM-L6-v2）已随安装包打包，无需下载。如果使用其他本地模型（BGE/E5/GTE 等），需要从 HuggingFace 下载，请确保网络能访问 `huggingface.co`。模型缓存在浏览器 IndexedDB 中，首次下载后不再需要网络。

**Q: 切换本地嵌入模型后检索无结果？**
A: 不同模型的向量维度不同（如 MiniLM 384维 vs BGE-zh 512维）。切换模型后，旧索引的向量维度与新查询不匹配，需要在笔记本管理中重新索引所有来源。控制台会输出维度不匹配的警告。

**Q: 本地嵌入模式索引很慢？**
A: 本地推理使用 WASM，速度比 API 慢。本地模式下索引批次已自动降为 10（远程为 100）。大型文档建议使用远程 API 模式。
